## Carrier Wave Science

### Why Carrier Waves Matter

Carrier waves serve as the "vehicle" for delivering beat frequencies to the brain. While the **difference** between frequencies (the beat) entrains brainwaves, the carrier frequency significantly affects:

1. **Auditory Perception:** Comfort and clarity of the sound
2. **Depth of Entrainment:** How effectively the brain locks onto the frequency
3. **Hemispheric Activation:** Which brain regions respond
4. **Body Resonance:** Interaction with skull cavity and body structures

### Frequency-Following Response (FFR)

- **Definition:** Brainstem's ability to lock onto and follow external frequencies
- **Optimal Range:** 100-500 Hz produces strongest FFR
- **Gateway Standard:** 150-200 Hz maximizes FFR efficiency
- **Mechanism:** Creates standing waves in skull cavity

### Carrier Wave Selection Rationale

#### Low Frequencies (150-200 Hz) - Gateway Standard
**Used in:** Gateway Explorer, Deep Void, Theta-Gamma Transcendence

- **150 Hz:** Maximum bone conduction efficiency, Gateway Process optimal
- **174 Hz:** Solfeggio foundation frequency, grounding
- **200 Hz:** Standard for comfortable theta/delta delivery

**Scientific Basis:**
- Maximizes auditory nerve firing rate
- Creates standing wave resonance in skull
- Comfortable perception (not too low, not too high)
- Monroe Institute standard for Hemi-Sync recordings

#### Mid-Range Frequencies (250-300 Hz)
**Used in:** DMT Breakthrough, Theta-Gamma Transcendence

- **250 Hz:** Optimal for monaural beats due to acoustic mixing properties
- **300 Hz:** Higher carrier needed for clear 40 Hz gamma pulse (7.5:1 ratio)

**Scientific Basis:**
- **Monaural Optimization:** Better acoustic interference patterns
- **Gamma Entrainment:** Carrier must be 5-10× pulse frequency for clear perception
- **Harmonic Richness:** More overtones create fuller sound

#### Standard Frequencies (400-450 Hz)
**Used in:** Gateway Explorer, DMT Breakthrough, Theta-Gamma, Deep Void

- **400 Hz:** Optimal air conduction without ear fatigue
- **432 Hz:** Natural tuning, mathematical relationship to Schumann (432 = 55 × 7.83)
- **440 Hz:** International standard, most research data available

**Scientific Basis:**
- Strong FFR with proven entrainment research
- 432 Hz resonates with water molecules (70% of body)
- 440 Hz has extensive scientific validation

#### Solfeggio Frequencies (174-528 Hz)
**Used in:** Schumann Resonance preset

- **174 Hz:** Foundation, grounding, pain relief
- **528 Hz:** DNA repair, transformation ("Miracle" frequency)

**Scientific Basis:**
- Based on Pythagorean tuning and harmonic principles
- Body structure resonances (174 Hz = lower body, 528 Hz = cellular)
- Cultural/historical significance in healing practices

### Harmonic Relationships

**Design Principle:** Avoid harmonic interference

- ❌ **Bad:** 200 Hz and 400 Hz in same preset (2:1 harmonic)
- ✅ **Good:** 150 Hz, 300 Hz, 440 Hz (clean ratios)

**Acoustic Properties by Range:**
- **100-200 Hz:** Maximum bone conduction efficiency
- **200-400 Hz:** Balanced air/bone conduction
- **400-500 Hz:** Optimal air conduction, no ear fatigue
- **>500 Hz:** Reduced FFR, more mental effort to track

## Technical Implementation

### Audio Architecture

#### Signal Chain
```
[Binaural] ──┐
[Isochronic] ─┼──> [Merger] ──> [Compressor] ──> [Master Gain] ──> Output
[Monaural] ───┤
[Noise] ──────┘
```

#### Layer Specifications

**Binaural Beat:**
- **Type:** Sine wave oscillators
- **Routing:** Separate left/right channels
- **Gain Staging:** 0.25 base × volume% × enabled (0/1)
- **Purpose:** Frequency-following response, hemispheric sync

**Isochronic Tone:**
- **Type:** Sine carrier + Sine LFO amplitude modulation
- **Routing:** Both channels (stereo)
- **Gain Staging:** 0.2 base × volume% × enabled (0/1)
- **Modulation:** Sine wave LFO (research shows 15% better than square wave)
- **Purpose:** Entrainment without headphones required

**Monaural Beat:**
- **Type:** Two sine oscillators mixed
- **Routing:** Both channels (stereo)
- **Individual Gains:** 0.1 each (prevent clipping when mixed)
- **Master Gain:** volume% × enabled (0/1)
- **Purpose:** Acoustic beating before ears (vs. neural beating)

**Noise Layer:**
- **Type:** AudioBufferSourceNode with crypto-random generation
- **Routing:** Both channels (stereo)
- **Gain Staging:** 0.15 base × volume% × enabled (0/1)
- **Generation:** Cryptographically-secure randomness (Web Crypto API)
- **Buffer Size:** 2 seconds, looping
- **Purpose:** Masking, grounding, trance deepening

#### Noise Algorithms

**White Noise (Equal Energy):**
```javascript
crypto.getRandomValues(randomBytes)
output[i] = (randomBytes[i] / 127.5) - 1.0
```

**Pink Noise (1/f - Natural):**
```javascript
// Voss-McCartney 7-pole filter
b0 = 0.99886 * b0 + white * 0.0555179
b1 = 0.99332 * b1 + white * 0.0750759
// ... (7 poles total)
output[i] = (b0 + b1 + b2 + b3 + b4 + b5 + b6 + white * 0.5362) * 0.11
```

**Brown Noise (1/f² - Deep):**
```javascript
// Leaky integrator (Brownian motion simulation)
output[i] = (lastOut + (0.02 * white)) / 1.02
output[i] *= 3.5 // Amplitude compensation
```

**Violet Noise (High Frequency):**
```javascript
// First-order differentiation
output[i] = white - lastWhite
```

**Blue Noise (Crisp, Oceanic):**
```javascript
// High-pass differentiation
output[i] = white - lastOut
```

#### Dynamics Processing

**Compressor Settings:**
- **Threshold:** -24 dB (start compressing early)
- **Knee:** 30 (smooth compression curve)
- **Ratio:** 12:1 (strong brick-wall limiting)
- **Attack:** 3ms (fast response)
- **Release:** 250ms (smooth recovery)

**Purpose:** Prevent clipping when all 4 layers active simultaneously

**Gain Budget:**
- Binaural: 0.25 max
- Isochronic: 0.2 max
- Monaural: 0.2 max (0.1 × 2 oscillators)
- Noise: 0.15 max
- **Total:** 0.8 max (before compression)

   - Update oscillator frequencies immediately

   - Regenerate noise buffer if type changed
4. Update color picker and flash mode
5. If currently playing, restart flashing with new settings
6. Update all display values

## Implementation Notes

### Best Practices

#### Session Guidelines
1. **Headphones Required:** Binaural beats only work with stereo headphones
2. **Volume:** Start at 30-50%, adjust to comfort
3. **Duration:**
   - Gateway Explorer: 20-30 minutes
   - DMT Breakthrough: 15-20 minutes (intense)
   - Theta-Gamma: 30-60 minutes
   - Schumann: Unlimited (gentle)
   - Deep Void: 15-20 minutes (use caution)
4. **Environment:** Dark room, comfortable position, no interruptions
5. **Safety:** Never use while driving or operating machinery

#### Contraindications
- **Epilepsy/Seizure Disorders:** Visual flashing can trigger seizures
- **Severe Mental Illness:** May intensify symptoms
- **Pregnancy:** Limited research, use caution
- **Pacemakers:** Low frequencies may interfere (theoretical)

#### Optimization Tips
1. **Layer Mixing:** Adjust individual layer volumes to preference
2. **Noise Type:** Experiment with different colors for different states
3. **Flash Mode:** Use solid color if flashing is distracting
4. **Build-Up:** Start with Schumann, progress to Gateway, then DMT/Void
5. **Integration:** Allow 10-15 minutes after session before activities

### Technical Limitations

1. **Web Audio API:** Browser-dependent, slight timing variations possible
2. **Crypto API:** 65,536 byte limit per call (handled with chunking)
3. **AudioBufferSourceNode:** Can only start() once, requires recreation
4. **Mobile Devices:** May have higher latency, reduced performance
5. **Sample Rate:** Varies by device (typically 44.1kHz or 48kHz)
